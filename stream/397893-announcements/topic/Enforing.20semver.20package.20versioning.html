Topic: Enforing semver package versioning

Luke Boswell (Jul 04 2026 at 01:58):

This is a notice for anyone who has a package (platform packages included) -- this PR https://github.com/roc-lang/roc/pull/9913 is planned when we are ready. Please begin to update any package release you have now.

  • Any published URL package (including platform packages) must now have a version of at least 1.0.0
  • Use roc bump to automatically see that the next version should be for your package
  • Roc will now diff the type checked API between versions and apply validation rules to mitigate breaking downstream packages.

Also -- please let me know if you experience any issues with this... I'll try to keep the branch rebased on latest main so you can build from source and test it.
