Topic: List.drop, dropFirst, dropLast

Anton (Oct 31 2023 at 12:42):

With today's nightly the following things have changed to the List builtins:

  • List.dropFirst now takes an extra argument with the number of elements to drop from the front.
  • Similarly List.dropLast now takes an extra argument with the number of elements to drop from the back.
  • List.drop has been renamed to List.dropFirst

So, to update your roc code:

  • change calls to List.dropFirst myList with List.dropFirst myList 1
  • replace all |> List.dropFirst with |> List.dropFirst 1
  • change calls to List.dropLast myList with List.dropLast myList 1
  • replace all |> List.dropLast with |> List.dropLast 1
  • replace all List.drop with List.dropFirst
