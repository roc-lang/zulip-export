In Roc core, the following have been implemented/merged:

  • Support for snake_case
    • snake_case names for builtins
    • roc format --migrate snake_case support
  • Support for PNC
    • PNC used in all builtins in the compiler (more or less)
    • roc format --migrate PNC support
  • Support of "${str}" interpolation in addition to the old "$(str)", with the plan to remove the old interpolation syntax eventually
  • Support for |args| ... in addition to the old \args -> ...
    • Unconditional roc format movement to new args syntax

There are two breaking changes remaining that we'd like to get in to Roc core:

  • support for zero-argument functions (e.g. lazy_load = || File.read!(...))
    • This feature will support us removing a lot of function calls with ({}), but will break a lot of code as a result.
    • Relatively important to merge before we finish this batch of changes
    • The parsing and formatting changes are all finished, but we are hitting a snag on specializing/monomorphizing zero-argument functions. This is pretty tricky, but I don't know how else to solve it except with a shovel and patience.
  • changing the syntax for open record destructures from { x, y } to { x, y, ... }
    • This will break every intentional usage of open record destructuring. We don't expect that's very much of the ecosystem, but this is an unavoidable break, unlike the ({}) to () change which is optional, but nice.
    • Not super important to merge before the next release, but nice to have
    • This is maybe 60-70% done, and is much more straightforward to implement than zero-arg functions, even if it's still tedious.
