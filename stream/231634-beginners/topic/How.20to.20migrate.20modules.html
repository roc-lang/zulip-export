Topic: How to migrate modules

Aurélien Geron (Jun 23 2026 at 00:24):

I'm looking into migrating some libraries to the new compiler, starting with roc-parser and unicode. Most of it looks fairly straightforward, but I'm not entirely sure how to reorganize modules. For example, in roc-parser, the Xml module currently starts like this:

module [Xml, XmlDeclaration, XmlVersion, Node, Attribute, xml_parser]

import Parser exposing [Parser, const, map, skip, keep, one_or_more, one_of, many, between, alt, chomp_while, flatten, lazy, chomp_until]
import String exposing [parse_str, string, Utf8, digits, codeunit_satisfies]

Xml : {
    xml_declaration : [Given XmlDeclaration, Missing],
    root : Node,
}

XmlDeclaration : {
    version : XmlVersion,
    encoding : [Given XmlEncoding, Missing],
}

XmlVersion := {
    after_dot : U8,
}
    implements [Eq]

v1_dot0 : XmlVersion
v1_dot0 = @XmlVersion(
    {
        after_dot: 0,
    },
)
...
expect ...
...
xml_parser : Parser Utf8 Xml
xml_parser =
    const(
        |xml_declaration|
            |root| {
                xml_declaration,
                root,
            },
    )
    |> keep(p_prolog)
    |> keep(p_element)
    |> skip(many(p_whitespace))
...

Should I migrate it to something like this?

import Parser exposing [Parser, const, map, skip, keep, one_or_more, one_of, many, between, alt, chomp_while, flatten, lazy, chomp_until]
import String exposing [parse_str, string, Utf8, digits, codeunit_satisfies]

Xml := {
    xml_declaration : [Given(XmlDeclaration), Missing],
    root : Node,
}. {

XmlDeclaration : {
    version : XmlVersion,
    encoding : [Given(XmlEncoding), Missing],
}

XmlVersion := {
    after_dot : U8,
}.{
    is_eq : XmlVersion, XmlVersion -> Bool
    is_eq = |v1, v2| v1.after_dot == v2.after_dot
}

v1_dot0 : XmlVersion
v1_dot0 = { after_dot: 0 }

...

xml_parser : Parser(Utf8, Xml)
xml_parser = {
    const(
        |xml_declaration| {
            |root| { {xml_declaration, root } }
        }
    )
    .keep(p_prolog)
    .keep(p_element)
    .skip(many(p_whitespace))
}
...
}

expect { ... }

Specifically:

  • The main type (Xml) must always be defined with := or ::. In this case, I've migrated it from a type alias to a type declaration. Is that correct?
  • implements [Eq] no longer exists so we must implement is_eq manually? Or is a new syntax coming?
  • The @XmlVersion syntax was removed, now we must rely on the type declaration, correct?
  • A function that returns a record must double the curly brackets? e.g. |root| { {xml_declaration, root } }
  • Any exposed function/type/constant must be part of the main type.
  • Since we're moving XmlVersion and XmlDeclaration inside of Xml, perhaps we should rename them to Version and Declaration to avoid the repetition?
  • Alternatively, should this module be split into multiple modules?

Jonathan (Jun 23 2026 at 08:52):

I can answer a couple I think

  • you should only need double brackets to disambiguate a punned record with a single element. If you have two it should be ok?
» f = |a, b| {a, b}
assigned `f`
» f(1, 2)
{ a: 1.0, b: 2.0 }
  • you need is_eq but I believe there will be a way to derive it for modules

Personal preference

  • I think it makes sense to drop the Xml prefix for nested modules, given it's already namespaced. Splitting into separate module files seems a bit too much for these imo.
