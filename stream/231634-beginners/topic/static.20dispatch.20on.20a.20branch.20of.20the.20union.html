Jonathan (Apr 09 2026 at 22:43):

Not to say whether or not that's clear from the error messages given, but I see it as something like:

  • If an expression is used as a statement, it must return unit. Therefore if it doesn't return unit, you have a type mismatch, which is an error.
  • Once the result has been assigned to a variable, there is no longer a type error, but the compiler is warning you that you haven't used it. This is different to not assigning at all.
