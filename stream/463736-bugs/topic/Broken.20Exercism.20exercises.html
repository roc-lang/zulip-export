Richard Feldman (Jul 22 2026 at 01:17):

I forget if this was discussed earlier in the thread, but the history here is:

  • the original Rust compiler did it the way we are currently doing it, where [Foo] is a closed tag union and [Foo, ...] is an open tag union, and only open tag unions can "grow"
  • a lot of people found this confusing, so later we introduced the "polarity" system where [Foo] means "open union if it's in an 'output position' (e.g. function return type) and closed if it's in an 'input position' (e.g. function argument)"
  • in practice, people seemed to find the "polarity" system less surprising/confusing, at the cost of it being the only place in the language where the same type annotation means something different depending on where it's used
  • since a bunch of things changed in the new compiler, we decided to defer the decision of whether to port over the polarity system to the new compiler - basically, see if it still seemed necessary after all or if we could get away with not having it and keep the type system simpler :smile:
