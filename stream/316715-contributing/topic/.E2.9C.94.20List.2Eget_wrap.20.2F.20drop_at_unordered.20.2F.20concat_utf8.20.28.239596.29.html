Dzmitry Misiuk (Jul 23 2026 at 17:25):

Almost everywhere it's clear:

  • Rust Vec::clear() — "no effect on the allocated capacity"
  • C# List<T>.Clear() — "Capacity remains unchanged"
  • C++ vector::clear()
  • Java ArrayList.clear()
  • Python list.clear()
  • JS Map.clear() / Set.clear()
  • Go clear() builtin
  • Zig clearRetainingCapacity() — the language we're in; ~500 uses in this codebase

Swift's the outlier: removeAll(keepingCapacity:). Couldn't find any language using emptied.
