Topic: error messages

Anton (Jul 01 2022 at 18:08):

To get excellent error messages it might be good to include a (preferably short) link with every error message where people can suggest improvements/shortcomings of that error. The link could prefill a github issue template and link to the line of code of the error. It would also be cool to include similar functionality in the editor for use with all roc packages. This could raise the error message quality of the whole roc ecosystem.
