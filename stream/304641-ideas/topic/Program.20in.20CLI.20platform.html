Richard Feldman (Oct 02 2022 at 06:53):

so originally the examples/cli-platform API for defining main was that you'd give it main : Task {} [] *

Richard Feldman (Oct 02 2022 at 06:56):

I changed it to be main : Program like Elm does, which I saw as having these tradeoffs:

  • it's more work to get up and running, because you have to call something like |> Program.noArgs in addition to making your Task
  • it's more conceptual overhead because you now have to learn another thing on top of Task
  • it makes it so that you can do things like getting the CLI arguments and environment variables as arguments to main instead of having to run Tasks to get them
  • it enables Program.quick : Task * * * -> Program which lets you write a CLI app that doesn't bother handling errors

Richard Feldman (Oct 02 2022 at 06:57):

I saw some feedback in #roctoberfest > Show and Tell that Program didn't feel great to use, and I'm curious how people feel about it

Richard Feldman (Oct 02 2022 at 07:01):

for example, an alternate design would be:

  • go back to main : Task {} [] *
  • have Arg.list : Task (List Str) * [Read [Args]*]* be the way to get CLI args (and probably also an Arg.walk and Arg.parseAll : Arg.Parser a -> Task a [ParseErr Arg.ParseErr]* [Read [Args]*]*), and tasks like Env.dict : Task (Dict Str Str) * [Env]* be the way to get environment variables
