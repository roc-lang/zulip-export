Richard Feldman (Jul 31 2024 at 01:00):

I don't know enough about wasm's memory model to be sure if this would work, but maybe in the wasm interop wrapper you could opt into some restrictions that gain performance without sacrificing security, specifically:

  • you give me some data structures like List etc.
  • as long as their refcount is 1 when you pass them in, they don't get copied, and I get direct access to their heap memory (with bounds checks unavoidably enforced somehow)
  • I can still heap allocate other memory if I want
  • no persistent state between calls, and no I/O access, so I don't have to be a Task
  • I can return pointers to the original memory, but the caller will automatically verify that pointer rules are being followed, refcounts are set accurately, etc.
