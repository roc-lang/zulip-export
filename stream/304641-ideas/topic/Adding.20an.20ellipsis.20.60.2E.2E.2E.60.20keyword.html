Topic: Adding an ellipsis `...` keyword

Sam Mohr (Dec 28 2024 at 23:30):

Our recent discussion on typechecking doc code blocks (#contributing > Ironing out doc code blocks) had a mention by Richard that he likes to put ... in teaching examples. It serves the purpose of allowing for obviously incomplete code used for explaining something in example code. An example usage would be:

update_file! = |path|
    contents = File.read!(path)?
    lines = contents.split_lines()

    updated_lines = lines.keep_oks(|line|
        updated = ... # your answer here

        Ok(updated)
    )

    new_path =
        if ... then ... else path

    File.write(path, updated_lines.join("\n"))

    Ok({})

This is a feature that sees success in Python. An ellipsis would function as a crash "not written yet" AST node, but unlike crash:

  • It will not tell code analysis that a crash has happened, meaning you won't get "unreachable code" warnings
  • It will produce a warning itself that says "This code is not finished yet" or something like that, which doesn't block running the code, but it blocks deploying it to production

This is a minor, but nice feature on its own. I think it becomes more desirable when we look at some of the other places it comes up:

  • In the #contributing > Ironing out doc code blocks discussion, we'd be able to require all codeblocks to at least typecheck, and also have no compiler errors. If we also add an allow_todos annotation, we could allow ellipses and then ensure there are no errors or warnings.
  • For #ideas > Inline type annotations, we can get empty function bodies to work without out-of-line type annotations by terminating function bodies with an ellipsis: read_file! = |path: Str| = ...

Some notes:

  • It was suggested that we maybe format ... to …, the Unicode character for an ellipsis. We don't currently do this anywhere in Roc, so unless we do it for other operators (maybe Unicode arrows)
  • I'm calling the ... a keyword rather than an operator since it's not used on anything, but is its own entity
