Richard Feldman (Nov 11 2024 at 19:36):

a way this could be taught:

First, teach that \val, arg -> val.foo(arg) has this inferred type:

val, arg -> ret
    where val.{ foo : val, arg -> ret }

Then, explain that in this syntax, where val.{ foo : val, arg -> ret } means:

  • The .{ foo : val, arg -> ret } part means that we're expecting a module to have a top-level foo function with this type
  • The val. at the front means "the module we expect to have that function is the module where val is defined"
    • If it turns out we can't access that module's top-level function from this scope, we get a compile-time error.
  • Method calls generate this inferred type by default, but you can also write it out by hand and use as to name the module where val is defined.
    • Once you've named that module, you can call any of the functions in it that the type specifies are there (e.g. the decode example).
