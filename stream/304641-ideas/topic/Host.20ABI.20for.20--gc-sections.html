Richard Feldman (Aug 07 2025 at 11:26):

given that we're already using lld for linking (with all the ergonomics pain points that come from host authors needing to provide a static .o file which we then give to lld to link into an executable), the delta if we want --gc-sections to work is:

  • host authors mark some functions as pub instead of passing pointers to them
  • the compiler tells llvm to use declare weak on the top-level functions it expects the app to get from the host
  • on Windows the compiler instead generates a .def file containing those same functions' types and then gives it to lld
  • for dev builds the compiler uses llvm to generate a struct of function pointers for these, since that's what the interpreter expects

Richard Feldman (Aug 07 2025 at 11:26):
