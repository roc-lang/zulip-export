Richard Feldman (Dec 24 2024 at 21:19):

in general, the philosophy behind warnings vs errors is:

  • both of them trigger nonzero exit codes, and "ignore warnings" is not a CLI flag by design, so the intention is that you'll fix all of them anyway
  • the distinction between warning and error is that an unfixed error may result in a runtime crash if you choose to run the program anyway, whereas a warning is guaranteed not to cause a runtime crash
  • since in this scenario we just ignore the extra field, it's not going to cause a runtime error, so it's a warning and not a carat crash

Richard Feldman (Dec 24 2024 at 21:20):

put another way, warnings are "will not affect end users in any way, but you definitely should clean this up" whereas errors may actually affect end users
