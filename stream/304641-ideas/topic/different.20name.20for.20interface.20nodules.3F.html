Richard Feldman (Oct 01 2022 at 09:01):

I wanted to try out the name "interface" instead of the obvious choice of "module" because:

  • these different types of .roc file all feel kinda like modules to me, so I wanted to give "the ones you can import" their own name - as opposed to (for example) saying "there are app modules, platform modules, and...normal modules" - but maybe a better design is to call the others something else other than modules?
  • I wanted to emphasize that creating a public interface (that is, choosing which operations the module exposes and which it hides) is the whole purpose of putting code in a module. I think the term "module" in the modern sense is often understood to be "a source code file with its own namespace," which is the least important aspect of a module. Unfortunately I don't think naming these "interfaces" actually helps further this educational goal much in practice.
