Topic: underscore prefix for unused type variable names

Richard Feldman (Jul 13 2025 at 03:17):

as of https://roc.zulipchat.com/#narrow/stream/316715-contributing/topic/Pull.20Request.20for.20Review/near/528497566 we no longer have the wildcard * type variable name. Instead, we generate type variable names like so:

empty_list : List(elem)
empty_list = []

reverse : List(elem) -> List(elem)

Previously, the empty list would have been typed List(*) instead.

Richard Feldman (Jul 13 2025 at 03:18):

One thing I liked about List(*) that the current design doesn't have is that you could tell at a glance (and also when talking out loud) whether a variable was bound (like in List(elem) -> List(elem) or unbound (like in List(elem) by itself where that's the whole type annotation. Notice how I felt the need to clarify just now?)

Richard Feldman (Jul 13 2025 at 03:18):

I had an idea: what if we used the underscore prefix for unbound type variables? e g.

empty_list : List(_elem)
empty_list = []

reverse : List(elem) -> List(elem)

Richard Feldman (Jul 13 2025 at 03:20):

the idea is that it's "unused" (which it is!) so we use that naming convention, whereas in reverse the two elem variables actually are used (by each other) so they don't get the underscore

Richard Feldman (Jul 13 2025 at 03:20):

I think this might be a way to get the benefits of * with the benefits of named vars at the same time
